## 3) 公式书写规则（强制遵守，含 HTML 渲染修复）
为了方便我复制到 PPT/文档，所有公式输出必须“去 LaTeX 化”，并且**修复 HTML 渲染残留的 LaTeX 控制符（尤其是 `\boldsymbol`）**，严格按以下规则清洗：

### 3.1 去 LaTeX 化输出规则
- 分数：`a/b`（不要 `\frac{a}{b}`）
- 下标上标：`x_i`、`x^2`（完全去掉 `{}`）
- 乘号：用 `*`（不要 `\times`、`\cdot`）
- 不要 `\left`、`\right`、不要花括号 `{}`、不要任何 LaTeX 控制符
- 保持可读性：必要时用括号 `( )` 明确优先级

### 3.2 HTML 公式渲染清理规则（必须修复）
- **HTML 中不允许出现任何形如 `\boldsymbol` 的残留**（以及同类 LaTeX 命令残留，如 `\mathbf`、`\mathit`、`\mathrm`、`\vec`、`\hat` 等）。
- 如果原书公式需要“粗体向量/矩阵”语义：
  - 在 `.md` 输出中用**纯文本约定**表达，比如（二选一，全文保持一致）：
    - 向量：`v (vector)` 或 `vec(v)`
    - 矩阵：`A (matrix)` 或 `mat(A)`
  - 在 `.html` 中必须保证最终显示**不包含 `\boldsymbol` 等 LaTeX 控制符**，并与 `.md` 一致。
- 对于已经生成的 HTML：
  - 必须对全站/全文件做一次系统性检查与替换，清除 `\boldsymbol` 及其导致的渲染错误，确保页面公式可读且样式一致。
- 修复完成后需在“修复日志”中标注：哪些文件/哪些公式段落涉及 `\boldsymbol` 清理，以及采用了哪种纯文本向量/矩阵表达约定。
